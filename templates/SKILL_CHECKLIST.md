# Skill authoring checklist

Use this before adding or updating any public skill.

## Scope
- The skill belongs to exactly one primary category.
- The skill composes cleanly with other categories.
- The skill avoids duplicating rules that belong elsewhere.

## Quality
- `name` and `description` are clear and short.
- The skill has a consistency contract.
- The skill defines a default workflow.
- The skill defines an output format.
- The skill defines safety rules.

## Public-readiness
- No internal company references.
- No internal file paths.
- No secret handling instructions.
- No destructive commands without confirmation.

## Composability
- Mentions which other skills it pairs with.
- Avoids conflicting rules with sibling skills.
- Keeps framework rules out of language-only skills.
